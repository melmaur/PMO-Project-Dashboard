# Project Charter

 
## PMO Project Dashboard – CRM Transformation Reporting Tool

---

## 1. Project Title  
PMO Project Dashboard – CRM Transformation Reporting Tool

---

## 2. Project Start Date  
January 2026 (simulated project timeline)

---

## 3. Project Purpose / Business Need  

Modern projects require clear visibility on progress, risks, and delivery status.

This project simulates a PMO reporting environment for a CRM implementation initiative, where project tracking is typically managed in Excel but lacks automation and visual insights.

The purpose of this project is to transform a static Excel project plan into an interactive dashboard that improves:

- Project visibility
- Executive reporting
- Task tracking
- RAID governance (Risks, Assumptions, Issues, Dependencies)

---

## 4. Objectives  

- Build an interactive PMO reporting dashboard using Python and Streamlit  
- Visualize project progress using KPIs and charts  
- Implement a Gantt-style project timeline  
- Introduce RAID log governance tracking  
- Improve clarity of project reporting for stakeholders  

---

## 5. Scope  

### In Scope:
- Project task tracking dashboard
- KPI reporting (progress, status counts)
- Gantt chart visualization
- RAID log module
- Interactive filters (Owner, Status, RAID Type)

### Out of Scope:
- Database integration
- Authentication system
- Real-time project management tools
- Integration with Jira / MS Project

---

## 6. Deliverables  

- Streamlit-based PMO dashboard application  
- Excel-based dataset for project tasks  
- RAID log tracker  
- GitHub repository with versioned development (V1–V4)  
- Project documentation and screenshots  

---

## 7. Stakeholders  

- Project Manager (simulated role)
- PMO Analyst (dashboard creator)
- Business Users (report consumers)

---

## 8. Success Criteria  

- Dashboard successfully displays project KPIs  
- Gantt chart accurately represents task timeline  
- RAID log is structured and filterable  
- Data can be updated via Excel file  
- Repository is well-documented and reproducible  

---

## 9. Assumptions  

- Data is maintained in Excel format  
- Users have basic understanding of project management concepts  
- Streamlit is used as the visualization framework  

---

## 10. Constraints  

- No enterprise database  
- No backend server  
- Local or lightweight deployment only  
- Simulated project data (not production system)  

---

## 11. Risks  

- Data inconsistency in Excel files  
- Misalignment between task dates and dependencies  
- Limited scalability beyond small datasets  

---

## 12. Project Completion Definition  

The project is considered complete when:

- Dashboard is fully functional  
- All PMO reporting components are implemented (KPIs, schedule, RAID)  
- Repository is documented and versioned (V1–V4)
